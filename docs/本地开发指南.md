# 本地开发指南

> 目标：clone 项目后，在本地跑起 **数据库 + Web**，能打开页面、上传照片、创建任务。  
> 配套：[技术方案](./技术方案-MVP全栈架构.md) · [开工包](./mvp-满月短片-开工包.md)

---

## 一、你需要安装什么

| 工具 | 版本建议 | 用途 | 安装方式 |
|------|----------|------|----------|
| **Docker Desktop** | 最新稳定版 | 本地 Postgres + Redis | [官网下载](https://www.docker.com/products/docker-desktop/) |
| **Node.js** | 20+ | 跑 Next.js | [nodejs.org](https://nodejs.org/) 或 `nvm install 20` |
| **pnpm** | 9+ | 包管理 | `npm install -g pnpm` |
| **Python** | 3.11+ | Worker 合成（Day 4 起） | 系统自带或 [python.org](https://www.python.org/downloads/) |
| **ffmpeg** | 任意近期版 | 视频合成（Day 4 起） | macOS: `brew install ffmpeg` |

**第一周最少要装：Docker、Node、pnpm。** Python 和 ffmpeg 在接 Worker 时再装也行。

## 二、Clone 后第一次启动（完整步骤）

### Step 1：Clone 代码

```bash
git clone <你的仓库地址> life-video-mvp
cd life-video-mvp
```

### Step 2：启动 Docker Desktop

打开 Docker Desktop，等菜单栏鲸鱼图标显示 **Running**（正在运行）。

验证：

```bash
docker --version
docker compose version
```

### Step 3：启动本地数据库

在项目**根目录**执行：

```bash
docker compose up -d
```

这会启动两个容器：

| 容器 | 作用 | 本机端口 |
|------|------|----------|
| `life-video-postgres` | PostgreSQL 数据库 | **5433** |
| `life-video-redis` | Redis（第二周队列用，现在可先跑着） | **6380** |

查看是否在跑：

```bash
docker compose ps
```

期望看到两个服务都是 `running`。

**常用命令：**

```bash
# 启动
docker compose up -d

# 停止（不删数据）
docker compose stop

# 停止并删除容器（数据卷保留，数据库数据还在）
docker compose down

# 看日志
docker compose logs postgres
```

### Step 4：配置 Web 环境变量

```bash
cd apps/web
cp .env.example .env
```

`.env` 默认内容（一般不用改）：

```bash
DATABASE_URL="postgresql://lifevideo:lifevideo@localhost:5433/life_video_mvp?schema=public"
STORAGE_ROOT="../../storage"
```

### Step 5：安装依赖并初始化数据库表

```bash
cd apps/web
pnpm install
pnpm db:migrate
```

`db:migrate` 会在 Postgres 里创建 `Job`、`JobAsset` 等表。  
如果提示输入 migration 名称，直接回车即可。

### Step 6：启动 Web

```bash
pnpm dev
```

浏览器打开：**http://localhost:3000**

## 三、验证是否跑通

### 3.1 数据库

```bash
# 根目录
docker compose ps
```

`life-video-postgres` 为 `running` 即可。

**可视化查看数据（网页界面）：**

```bash
cd apps/web
pnpm db:studio
```

浏览器打开 **http://localhost:5555**，可查看、编辑 `Job`、`JobAsset` 表。  
（需先 `docker compose up -d` 且已执行过 `pnpm db:migrate`）

也可直接用 psql（可选）：

```bash
docker exec -it life-video-postgres psql -U lifevideo -d life_video_mvp -c '\dt'
```

应能看到 `Job`、`JobAsset` 等表。

### 3.2 Web 页面

| 地址 | 预期 |
|------|------|
| http://localhost:3000 | 首页，有「开始制作 · 宝宝满月」 |
| http://localhost:3000/create/baby-moon | 创作页，可上传照片、填表单 |
| 提交后 | 跳转到 `/jobs/xxx`，显示「排队中」 |

> **说明：** Day 4 Worker 已接入。需同时跑 `python main.py`，任务才会从「排队中」变为「生成成功」。

### 3.3 全链路测试（上传 → 合成 → 下载）

1. 三个终端分别跑：**Docker**、`pnpm dev`、`python main.py`（见第四节）
2. 准备 **5～15 张** JPG/PNG 照片
3. 打开 `/create/baby-moon`，填写信息并提交
4. 跳转到 `/jobs/xxx`，应看到进度：排队中 → 正在合成 → 生成成功
5. 页面出现 `<video>` 预览，可点击下载 MP4

成片文件在：`storage/outputs/{jobId}.mp4`

## 四、以后每天怎么启动（已配置过一次）

开 **三个终端**（全链路跑通时）：

**终端 1 — 基础设施**

```bash
cd life-video-mvp
docker compose up -d
```

**终端 2 — Web**

```bash
cd life-video-mvp/apps/web
pnpm dev
```

**终端 3 — Worker（Day 4，合成视频）**

```bash
cd life-video-mvp/worker
cp .env.example .env    # 首次
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python main.py
```

Worker 启动后会轮询 `created` 任务，合成完成后任务页可预览、下载 mp4。

## 五、Worker + ffmpeg（Day 4 起，可选提前装）

合成视频需要 Python Worker 和 ffmpeg。

### 5.1 安装 ffmpeg

```bash
brew install ffmpeg
ffmpeg -version
```

### 5.2 Python 环境

```bash
cd worker
cp .env.example .env
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python main.py
```

Worker 的 `.env`（从 `.env.example` 复制）：

```bash
DATABASE_URL="postgresql://lifevideo:lifevideo@localhost:5433/life_video_mvp?schema=public"
STORAGE_ROOT="../storage"
PROJECT_ROOT=".."
WORKER_POLL_INTERVAL=3
```

> Worker 会自动去掉 `DATABASE_URL` 里的 `?schema=public`（Prisma 专用参数，psycopg2 不支持）。

### 5.3 背景音乐

仓库里 `assets/bgm/` 默认是空的（版权原因不进 git）。合成时需要 BGM 文件：

```text
assets/bgm/
  warm_piano.mp3
  soft_guitar.mp3
  lullaby.mp3
```

可从 [Pixabay Music](https://pixabay.com/music/) 等无版权站点下载，按 preset 名命名放入即可。  
本地调试也可临时把 mp3 放到 `test-assets/bgm/`，`render.py` 会 fallback 读取。

## 七、常见问题

### Docker 启动失败：`Cannot connect to the Docker daemon`

→ Docker Desktop 没打开。先启动 Docker Desktop，等 Running 后再执行 `docker compose up -d`。

### `pnpm db:migrate` 连不上数据库

1. 确认 `docker compose ps` 里 postgres 是 running  
2. 确认 `apps/web/.env` 里端口是 **5433**（不是 5432）  
3. 刚启动 Docker 后等几秒再 migrate

### 端口 5433 被占用

改 `docker-compose.yml` 里 postgres 的端口映射，例如 `"5434:5432"`，并同步修改 `.env` 里的 `DATABASE_URL`。

### 提交任务报 500

1. 看终端里 `pnpm dev` 的错误日志  
2. 确认已执行 `pnpm db:migrate`  
3. 确认 `storage/uploads/` 目录存在且可写

### 任务一直「排队中」

→ 检查 **Worker 是否在跑**（`cd worker && python main.py`）。只有 Web 没有 Worker，任务会停在 `created`。

### 换了一台新电脑

按本文 **第二节「Clone 后第一次启动」** 从头做一遍即可。  
数据库数据在 Docker volume 里，**换电脑不会自动迁移**，需要重新 `db:migrate`；旧任务数据不会跟过来。

## 八、项目目录（和启动相关）

```text
life-video-mvp/
├── docker-compose.yml      # 数据库、Redis — 根目录 docker compose up -d
├── apps/web/               # Next.js — pnpm dev
│   ├── .env                # 本地配置（从 .env.example 复制，不提交 git）
│   └── prisma/             # 数据库表结构
├── worker/                 # Python 合成 — python main.py（Day 4）
├── templates/              # 视频模板 JSON
├── assets/bgm/             # 背景音乐（需自行下载放入）
└── storage/                # 上传素材与成片（运行时生成，不提交 git）
```

## 九、相关命令速查

```bash
# === 根目录 ===
docker compose up -d          # 启动数据库
docker compose ps             # 查看状态
docker compose stop           # 停止

# === apps/web ===
pnpm install                  # 安装依赖
pnpm db:migrate               # 同步数据库表结构
pnpm db:push                  # 仅推送 schema（无 migration 文件时用）
pnpm dev                      # 开发服务器
pnpm build                    # 生产构建检查
pnpm lint                     # Biome 检查

# === worker（Day 4）===
python main.py                # 启动 Worker
python render.py              # 单独测试 ffmpeg 合成
```
