## 1. 核心结论（先看这个）

你的处境比从零做 [HudyAI](https://hudyai.com/) 或 [即梦](https://jimeng.jianying.com/) 竞品要好得多：

- 你已做过 [奇觅](https://qimi.com/) 前端，懂视频产品的上传 → 配置 → 生成 → 下载流程
- 有朋友真实需求（满月、结婚纪念等生活化剪辑），且你帮人工剪过
- 公司后端/算法你不必复刻，全链路用 **「模板 + ffmpeg + 任务流」** 即可落地

**最佳策略：**

```text
不要：复刻奇觅算法 / 做即梦竞品 / 做 AI 全家桶
要做：生活场景 1 分钟短片工具（满月先做，结婚纪念第二版）
技术：Next.js + 队列 + Python/ffmpeg + OSS
AI：可选点缀，不是核心
```

## 2. 三类产品，其实是三种不同问题

把奇觅、即梦、朋友需求拆开看，就不会觉得「什么都得做」：

### 2.1 奇觅（游戏广告 AI 成片）

参考：[奇觅 — 游戏广告 AI 制作平台](https://qimi.com/)

| 维度 | 说明 |
|------|------|
| 核心 | 上传游戏录屏 + BGM → AI 脚本、结构编排、表情包混剪 |
| 产出 | 单人每天混剪上百条可投放短片 |
| 用户 | 投放人员 / 游戏运营，偏 B 端 |
| 素材 | 游戏录屏，结构相对固定 |
| 算法 | 公司内部模型与编排（非个人可复刻） |

### 2.2 即梦（文/图生短视频）

参考：[即梦 AI](https://jimeng.jianying.com/)

| 维度 | 说明 |
|------|------|
| 核心 | 输入文案或图片 → AI **生成** 5～10 秒创意片段 |
| 用户 | 创作者，要灵感与视觉效果 |
| 特点 | 片段短、创意强，**不等于完整叙事视频** |
| 成本 | 按次/API 计费，视频类偏贵 |

### 2.3 朋友要的（满月 / 结婚纪念 / 生活纪念）

| 维度 | 说明 |
|------|------|
| 核心 | 一堆照片 + 零散视频 → **1 分钟左右**、有音乐、字幕、情绪 |
| 用户 | 普通消费者，要「帮我剪好」 |
| 形态 | 像剪映模板、婚礼相册、店铺宣传短片 |
| 本质 | **模板化智能剪辑**，不是纯文生视频 |

### 对比一览

|  | 奇觅 | 即梦 | 生活短片（建议你做） |
|--|------|------|---------------------|
| 时长 | 短、偏广告 | 5～10 秒/段 | 30～90 秒成片 |
| 素材 | 游戏录屏 | 文/图生成 | 用户照片+视频 |
| 逻辑 | AI 混剪+投放 | AI 生成片段 | **模板+规则编排** |
| 难度（个人） | 高（依赖算法团队） | 中高（API+成本） | **中（ffmpeg+模板）** |
| 真实需求 | 公司内部 | 泛创作者 | **你朋友已验证** |

## 3. 要不要复刻奇觅？和公司项目什么关系？

### 3.1 要不要做「奇觅完整复刻版」？

**不要。也不应该。**

| 公司里 | 个人全链路时 |
|--------|-------------|
| 奇觅前端 | 可借鉴交互与流程理解 |
| 公司后端/算法/大模型 | **不能直接用公司代码** |
| 游戏广告混剪 | 朋友要的是生活化 1 分钟视频 |

全链路 = **你自己负责从需求到上线**，不是一个人重写算法团队能力。

你要补的是：

```text
需求定义 → 简化版后端 → 任务编排 → 现成能力（ffmpeg / 模板 / 可选 API）→ 存储 → 部署
```

**算法层 = 「编排 + 模板 + 拼接」，不是自研大模型。**

### 3.2 可以借鉴 vs 不能照搬

**可以借鉴：**

- 奇觅的产品流程（上传 → 配置 → 生成 → 等待 → 下载）
- 视频任务等待体验、进度展示、错误提示
- 素材限制（格式、大小、数量）等产品细节

**不能照搬：**

- 公司后端代码与游戏混剪算法
- 游戏投放导向的模板与脚本逻辑

## 4. 为什么即梦 5～10 秒解决不了 1 分钟需求？

朋友要的不是「一个 AI 镜头」，而是**完整叙事**：

```text
开场标题 → 照片回忆 → 几段视频 → 祝福语字幕 → 结尾致谢 → 配 BGM
```

### 错误思路

用即梦类 API **一次性生成 1 分钟 AI 视频** → 成本极高、效果不稳定、情感假。

### 正确思路：拼装式成片

```text
用户上传素材（照片/视频）
    ↓
选场景模板（满月 / 结婚纪念 / 生日…）
    ↓
自动裁切、排序、加字幕、加转场、加 BGM
    ↓
（可选）插入 1～2 段 AI 生成的 5 秒空镜/装饰
    ↓
ffmpeg 合成一条 60 秒成片
```

**即梦适合做「点缀」，不适合单独扛 1 分钟生活视频。**

## 5. 你应该做什么产品？

### 5.1 产品定位

> **「生活场景 AI 短片助手」**  
> 帮普通人用照片/视频，几分钟做出 1 分钟纪念/宣传短片

### 5.2 项目名称

**`life-video-mvp`**（生活场景纪念短片）

### 5.3 模板路线（分阶段）

**第一版（当前）：**

1. **宝宝满月纪念**（约 60 秒）

**第二版（满月全链路跑通后）：**

2. **结婚纪念短片**（约 60～90 秒）

**第一版功能：**

- [ ] 上传 5～20 张照片 + 0～3 段短视频
- [ ] 选模板
- [ ] 填关键信息（名字、日期、祝福语）
- [ ] 选 BGM（内置 5 首）
- [ ] 一键生成
- [ ] 预览 + 下载

**第一版不做：**

- 文生视频大平台、即梦竞品
- 自由时间轴剪辑（剪映 Pro 级）
- 游戏广告混剪
- 复杂 AI 脚本生成

### 5.4 模板时间轴示例（满月 60 秒）

```text
0-5s   ：标题卡（宝宝名 + 日期）
5-40s  ：照片轮播，每张约 3 秒，淡入淡出
40-55s ：视频片段（有则插入，无则跳过）
55-60s ：结尾祝福语 + 配乐渐出
```

可用 **ffmpeg + 模板 JSON** 实现，无需训练模型。

## 6. 和奇觅比，是不是更简单？

**是的，对个人全链路而言简单很多。**

| 维度 | 奇觅（游戏广告） | 生活视频 MVP |
|------|----------------|--------------|
| 素材 | 游戏录屏，较固定 | 照片+短视频，更杂但规则可模板化 |
| 成片逻辑 | AI 脚本+混剪+投放 | **模板驱动 + 规则编排** |
| 时长 | 短、偏广告 | 30～90 秒 |
| 算法 | 公司内部 | **ffmpeg + 模板，少量 AI 可选** |
| 后端复杂度 | 高 | 中等 |
| 成本 | 高 | **低很多** |
| 你的熟悉度 | 前端流程熟 | 朋友需求真实 |

## 7. 全链路技术架构

### 7.1 要不要 Java？要不要 Python？

| 层级 | 建议 | 说明 |
|------|------|------|
| 前端 | Next.js / React | 本职，上传/配置/进度/预览 |
| 主 API | **NestJS 或 Next.js API** | MVP 不必 Java |
| 视频合成 Worker | **Python + ffmpeg**（推荐） | 裁切、拼接、字幕、配乐生态好 |
| 大模型/生成 | **调第三方 API（可选）** | 不自研 |
| 任务队列 | Redis + BullMQ / Celery | 异步合成必做 |
| 存储 | 阿里云 OSS / 腾讯云 COS | 素材与成片 |

### 7.2 架构图

```text
用户浏览器
    ↓
Next.js 前端
  - 上传照片/视频
  - 选模板、填文案、选音乐
  - 进度、预览、下载
    ↓
API 服务（NestJS / Next API）
  - 用户 / 任务 / 模板
  - 创建 render job
  - 查询状态
    ↓
任务队列（Redis + BullMQ）
    ↓
Worker（Python 推荐）
  - 下载用户文件
  - 按模板时间轴 ffmpeg 合成
  - （可选）调 1 次图/视频 API 做空镜
  - 上传成片到 OSS
    ↓
CDN / 签名 URL 给用户下载
```

第一版可收敛为：

```text
Next.js 全栈 + Redis 队列 + 1 个 Python worker + OSS
```

### 7.3 核心数据表

```text
users           用户
templates       模板定义（JSON 时间轴）
jobs            任务（status: pending / processing / success / failed）
job_assets      用户上传的素材
job_outputs     成片文件
credits         积分/次数（可选）
```

### 7.4 任务状态机

```text
created → queued → preprocessing → rendering → uploading → completed
                                                    ↘ failed
```

前端轮询或 WebSocket 展示进度（你在奇觅里已有类似经验）。

### 7.5 你要写的「算法」是什么？

不是 diffusion，而是**剪辑规则 + 模板 JSON**，例如：

```json
{
  "name": "baby_full_moon",
  "duration": 60,
  "segments": [
    { "type": "title", "start": 0, "duration": 5, "fields": ["babyName", "date"] },
    { "type": "slideshow", "start": 5, "duration": 35, "photoDuration": 3, "transition": "fade" },
    { "type": "video_clips", "start": 40, "duration": 15, "optional": true },
    { "type": "ending", "start": 55, "duration": 5, "fields": ["blessing"] }
  ],
  "audio": { "bgm": "warm_piano.mp3", "fadeOut": 2 }
}
```

## 8. 怎么把短 AI 片段和 1 分钟成片结合？

AI 作**配角**，主体仍是用户素材：

| 用途 | 做法 |
|------|------|
| 开场/结尾空镜 | 即梦/通义等 API 生成约 5s 温馨背景 |
| 转场装饰 | 抽象光效、花瓣等短片段 |
| 照片不够 | 文生图补 1～2 张氛围图（可选） |

这样：成本低、时长够、情感更真、不像纯 AI 假视频。

## 9. AI 工具平台（HudyAI 类）补充说明

若未来做 [HudyAI](https://hudyai.com/) 那种「图/音/视频工具矩阵」，难度高于生活短片 MVP：

| 难点 | 生活短片 MVP | HudyAI 级平台 |
|------|-------------|---------------|
| 模型费用 | 可几乎不用 AI | 按次烧 API |
| 异步任务 | 需要 | 需要且更复杂 |
| 自研算法 | 不需要 | 不需要，但接入模型多 |
| 适合个人第一步 | **是** | 否（先单品） |

个人路径建议：

```text
生活模板短片（你现在） → 单点 AI 能力（如照片风格化） → 再考虑多工具平台
```

### 若做 HudyAI 类单品（参考）

- 图片风格化、人声分离比「全长视频生成」更适合先做
- 后端：NestJS + 队列 + OSS + Replicate/fal/国内 API
- 不必 Java；媒体处理可用 Python worker

## 10. 成本估算

### 生活短片 MVP（模板为主，不调 AI）

| 项目 | 量级 |
|------|------|
| ECS | ¥100～300/月 |
| OSS + CDN | ¥几十/月 |
| AI API | ¥0（第一版可不加） |
| 合成 | 主要 CPU，可控 |

### 若每条加 1～2 次 AI 空镜

| 项目 | 量级 |
|------|------|
| 单次短视频 API | 约 ¥1～10 |
| 建议 | 免费用户不加 AI，付费再加 |

产品层：**免费次数 + 订阅/按条付费**，避免无限免费视频生成。

## 12. 面试叙事模板

### 不要这样说

「我把公司奇觅又做了一遍。」  
「我仿了即梦。」

### 推荐说法

> 「我在公司参与了 [奇觅](https://qimi.com/) 游戏广告 AI 视频平台的前端。之后发现身边朋友有满月、结婚纪念等生活化短片需求，和公司场景不同。我独立做了 `life-video-mvp`，负责需求、前后端和视频合成链路，用模板编排 + ffmpeg 生成约 1 分钟成片，而不是自研大模型。」

### 五个故事（各 1～2 分钟）

1. **为什么做**：朋友需求 vs 奇觅/即梦差异  
2. **需求怎么砍**：为什么先做满月、后做结婚纪念，不做自由剪辑  
3. **技术取舍**：为什么模板+ffmpeg，为什么 Python worker，为什么不用 Java  
4. **线上问题**：合成失败、大文件、超时怎么处理  
5. **AI 边界**：第一版为什么可以不加 AI，加的话只做空镜

### 简历项目描述（可改）

```
【项目名称】life-video-mvp — 生活场景纪念短片

【背景】朋友有宝宝满月、结婚纪念等 1 分钟短片需求；公司奇觅面向游戏广告，场景不同。

【职责】独立负责需求、前端、API、任务队列、ffmpeg 合成脚本、OSS 部署。

【技术栈】Next.js + PostgreSQL + Redis + BullMQ + Python/ffmpeg + 阿里云 OSS

【成果】
- 场景模板：满月（已上线）+ 结婚纪念（第二版），用户上传素材一键生成 45～90 秒成片
- 异步任务与进度展示，支持预览与下载
- 已为 X 位朋友制作 X 条真实视频并迭代模板

【亮点】理解游戏广告 AI 产品与 C 端生活剪辑的差异；全链路可演示、可讲清架构与取舍。
```

## 13. 核心问题速查表

| 问题 | 答案 |
|------|------|
| 奇觅方向是不是更熟悉？ | 流程熟悉，但**不要复刻算法**，换生活场景更简单 |

| 朋友要生活剪辑，做游戏还是生活？ | **生活化**；需求真实，且与公司产品不冲突叙事 |
| 即梦 5～10 秒，朋友要 1 分钟？ | **模板拼装 60 秒**，AI 只点缀 |
| 要不要 Java？ | MVP **不必** |
| 要不要自研大模型？ | **不要** |
| 算法用什么？ | **规则 + 模板 JSON + ffmpeg**；可选 Python worker |
| 能练全链路吗？ | **能**，且很适合当前背景 |
